# 代理模型预测问题树 (Prediction Problem Tree)

## 0 根节点 (Root Problem)
**现象描述**：早期代理模型在测试集上表现出严重的“均值坍缩（Mean Collapse）”现象，预测分数高度集中在均值附近，差异极小。点预测误差（MAE/RMSE）掩盖了模型无法正确区分样本优劣的本质，导致强迫选择排序准确率（Pairwise Ranking Accuracy）极低（接近 0.5 的随机水平）。

## 1 第一层归因分析 (Level 1 Analysis)
针对均值坍缩和排序失效，梳理出三个主要维度的潜在原因：

* **节点 A：模型设计复杂度过高 (Architecture Complexity)**
    * 过多复杂的物理建模模块（如 Road Transformer, DeepSets 聚合层等）导致反向传播时梯度消失或信息瓶颈。模型无法有效提取高阶特征，退而求其次选择输出均值以获得最安全的局部最优解。
* **节点 B：损失函数约束错位 (Loss Function Misalignment)**
    * 过度依赖传统的高斯假设 MSE 损失，导致模型极易受到极端异常值（Outliers）的拉扯，丧失了对主体数据分布刻画的鲁棒性。
* **节点 C：数据分布与表征限制 (Data Representation)**
    * 底层纯物理特征（如距离、排队长度）可能不足以支撑模型理解复杂的路网协同逻辑；同时，前期 320 条数据的小样本规模导致模型极易过拟合。

## 2 第二层：解决方案分支 (Level 2 Branches)

根据上述归因，我们制定了相应的改进分支路线：

### [分支 1] 模型架构极简重构 (Branch 1: Architecture Simplification) 
* **核心动作**：全面废弃底层物理特征的手动拼接与复杂的图/集合聚合网络。
* **解决方案**：引入大型语言模型（如 Qwen2.5）的先验知识，直接截取其高维语义隐状态（`hidden_states`），外接极简的两层多层感知机（MLP）进行降维与分数映射。
* **状态**：✅ **已验证成功**。详见 `branch1.md`。

### [分支 2] 损失函数鲁棒性升级 (Branch 2: Loss Function Robustness)
* **核心动作**：放弃 MSE，转向支持长尾分布的概率模型。
* **解决方案**：采用 Student-t 负对数似然损失（NLL），输出 $\mu, \sigma, \nu$。利用动态的置信度 $\sigma$ 自适应抵消极端样本的梯度拉扯。
* **状态**：✅ **已在主干代码中集成**。

### [分支 3] 数据规模与质量验证 (Branch 3: Data Scaling & Quality)
* **核心动作**：排查数据同质化问题，扩大样本多样性。
* **解决方案**：利用并行生成的批量脚本，调整 LLM 的采样温度（如 `temperature=0.7`），将数据集从 320 条无监督扩充至 5000 条规模，彻底解决绝对刻度漂移问题（提升 $R^2$ 指标）。
* **状态**：🔄 **待下一步执行**。
