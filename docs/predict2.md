# 鲁棒的分布预测奖励模型：核心训练方案

## 一、 目标
训练一个具备不确定性感知能力的奖励模型。模型不仅预测目标得分，还能输出对该预测的置信度（方差），从而在 PPO 强化学习阶段有效抵御离群点带来的过度拟合。

---

## 二、 输入
在时间步 $t$，模型接收以下拼接特征序列：
* **$\tau$：** 大模型推理产生的中间隐状态序列。
* **rewards：** 过程奖励模型（PRM）给出的步骤级奖励序列。

---

## 三、 输出
模型放弃输出单一标量，转而输出目标分布（推荐 Student-t 分布）的参数：
* **$\mu$：** 预测的期望得分（均值）。
* **$\sigma^2$：** 预测的不确定性大小（方差，实际工程中通常输出对数方差 $\log(\sigma^2)$ 以防数值爆炸）。
* **$\nu$：** 分布的自由度参数（控制尾部厚度，容忍极端值）。

---

## 四、 损失函数
使用 **Student-t 负对数似然损失 (NLL)** 替代传统的均方误差 (MSE)。它能自适应调节惩罚力度，对种群进化中常见的极端极差/极优突变体具备极强的鲁棒性。

核心公式：
$$\mathcal{L}(\theta) = -\log \left( \frac{\Gamma(\frac{\nu+1}{2})}{\Gamma(\frac{\nu}{2}) \sqrt{\nu \pi} \sigma} \left( 1 + \frac{(\rho - \mu)^2}{\nu \sigma^2} \right)^{-\frac{\nu+1}{2}} \right)$$

---

## 五、 训练流程
1. 加载 population 数据，提取每个 individual 的 $\tau$、rewards、真实得分 $\rho$。
2. 按 group（如 Prompt 级别）划分 train/valid/test 集，绝对避免数据泄漏。
3. 只使用训练集 (train) 来拟合数据的标准化参数（均值与方差）。
4. 训练网络模型，以优化（最小化） Student-t NLL 或 Mixture NLL 为目标。
5. 在验证集 (valid) 上持续监控 NLL、CRPS、MAE、coverage 和 rank correlation 等多维指标。
6. 触发 early stopping 机制，选择验证集 NLL 最低（最优）的模型权重进行保存。
7. 在完全独立的测试集 (test) 上运行最优模型，报告最终的泛化性能。
