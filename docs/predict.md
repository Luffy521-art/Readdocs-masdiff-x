# 分布预测模型：不确定性感知的奖励预测训练方案

## 一、 模型训练目标

传统奖励模型（Reward Model）通常只输出一个标量分数（Point Estimate），这在强化学习中容易导致“过度自信”的错误。本模型的训练目标是实现不确定性感知的奖励分布预测（Uncertainty-Aware Reward Prediction）。

具体而言：

* **输入端：** 接收一段完整的生成序列，该序列由每一步骤的“内部语义认知”（模型隐状态 $\text{Hidden_State}_t$）和“外部规则评价”（过程奖励 $\text{PRM}_t$）拼接构成。
* **输出端：** 放弃输出单一标量 $\hat{\rho}$，转而输出一个概率分布的参数。假设真实奖励 $\rho$ 服从高斯分布，模型的目标是精准输出该分布的均值 $\mu$（期望得分）与方差 $\sigma^2$（模型对该得分的不确定度）。

> **核心价值：** 在 PPO 训练阶段，当模型遇到方差 $\sigma^2$ 极大的预测时，算法可以据此降低信任度，或者作为内在奖励（Intrinsic Reward）鼓励模型去探索该未知领域。

---

## 二、 损失函数

为了训练模型输出正确的分布参数，我们必须摒弃传统的均方误差（MSE），转而使用负对数似然损失函数（Negative Log-Likelihood, NLL）。
其物理意义是：最大化模型预测的分布在真实得分 $\rho$ 处的概率密度。

### 1. 理论公式
假设模型输出的均值为 $\mu$，方差为 $\sigma^2$，真实标签为 $\rho$，单个样本的高斯 NLL 损失函数为：

$$\mathcal{L}(\theta) = \frac{1}{2} \log(2\pi\sigma^2) + \frac{(\rho - \mu)^2}{2\sigma^2}$$

### 2. 工程实现公式（数值稳定性优化）
在实际写代码时，直接让神经网络预测方差 $\sigma^2$ 非常危险，因为网络输出可能是负数，导致对数爆炸。标准做法是让模型预测对数方差 $v = \log(\sigma^2)$。此时损失函数转化为无约束的平滑版本：

$$\mathcal{L}(\theta) = \frac{1}{2} \left( v + \frac{(\rho - \mu)^2}{\exp(v)} \right) + C$$

*(注：常数项 $C = \frac{1}{2}\log(2\pi)$ 在计算梯度时会被消除，代码实现中可省略。)*

### 3. 损失函数的自适应博弈机制
* **当模型预测准确时：** $(\rho - \mu)^2$ 很小，模型为了进一步降低 Loss，会主动缩小 $v$（即减小方差，表现出“自信”）。
* **当模型预测严重失误时：** $(\rho - \mu)^2$ 很大，除以 $\exp(v)$ 会导致误差项爆炸。为了自救，模型会大幅提高 $v$（即增大方差，承认“我不确定”），从而有效防止因过度拟合离群点（Outliers）而导致的网络崩溃。

## 三、 评估指标

评估一个分布预测模型，需要从绝对精度和不确定性质量两个维度进行。

### 1. 精度指标（看 $\mu$ 是否准）
* **均方误差 (MSE) / 平均绝对误差 (MAE)：** 仅计算预测均值 $\mu$ 与真实标签 $\rho$ 之间的物理距离。
* **斯皮尔曼等级相关系数 (Spearman Rank Correlation)：** 衡量排序能力。对于 PPO 而言，模型能不能把好的回答排在坏的回答前面，比绝对分数更重要。

### 2. 分布/不确定性指标（看 $\sigma^2$ 是否合理）
* **测试集负对数似然 (Test NLL)：** 这是最综合的指标。Test NLL 越低，说明模型不仅均值猜得准，对自己的信心也评估得很客观。
* **误差-不确定性相关性 (Error-Uncertainty Correlation)：** 取测试集样本计算绝对误差 $|\rho - \mu|$ 和预测标准差 $\sigma$ 的皮尔逊相关系数。优秀的分布模型，其相关系数应强正相关——预测错得越离谱的地方，模型报的 $\sigma$ 就应该越大。

## 四、 训练流程

### 1. 严格的数据划分 (Data Split)
这是防止强化学习数据泄露的红线。必须按 Prompt 级别进行 K 折交叉验证或划分。
如果 Prompt_A 被分入训练集，那么 Actor 模型针对 Prompt_A 生成的所有 5 个 Responses，以及对应的隐状态和真实 $\rho$，必须全部留在训练集中。严禁同一道题的两个变体分别出现在训练集和测试集中。

### 2. 数据预处理与特征拼接 (Feature Engineering)
在时间步 $t \in [1, L]$，提取特征：
* $h_t$：Qwen 最后一层的 Hidden State（例如 3584 维）。
* $prm_t$：PRM 给出的该步过程奖励（标量）。

将二者拼接：$x_t = [h_t; prm_t]$。每个样本构成形状为 $(L, D_{\text{hidden}} + 1)$ 的序列矩阵。

### 3. 网络搭建 (Architecture)
* **序列聚合层 (Aggregator)：** 使用 1D-CNN 提取局部逻辑特征，或直接使用 Mean-Pooling + 最后一步输出（Last Token Representation），将 $L$ 维度的序列压缩为固定长度的上下文向量 $C$。
* **分布投影头 (Distribution Head)：** 将 $C$ 送入两层 MLP。MLP 的最后一层分为两个独立线性映射：分支一输出标量 $\mu$（无激活函数）；分支二输出标量 $v$（即 $\log(\sigma^2)$，无激活函数）。

### 4. 模型训练 (Training Loop)
* **前向传播：** 批量输入特征矩阵，输出 $\mu$ 和 $v$。
* **计算损失：** 使用上述的 NLL Loss 公式。
* **反向传播与优化：** 使用 AdamW 优化器。建议开启梯度裁剪（Gradient Clipping），因为在训练初期，模型极度不自信（$v$ 剧烈震荡）时，NLL Loss 容易产生巨大的梯度。
* **早停机制 (Early Stopping)：** 监控验证集上的 Test NLL。当 Test NLL 连续若干个 Epoch 不再下降时终止训练，防止模型对方差的预测过拟合。
